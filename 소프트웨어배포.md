# 🎯 **1. 설치형(Installer 기반 배포)**

Windows 소프트웨어 배포에서 가장 널리 쓰이는 방식.

### ✔ 종류

* **MSI 설치 파일**
* **EXE 설치 프로그램** (Inno Setup, NSIS, InstallShield 등)
* **MSIX 패키지** (현대적, WinUI3 공식 권장)

### ✔ 특징

* 바탕화면/시작 메뉴 등록
* 레지스트리 설정 가능
* 언인스톨러 자동 생성
* 기업 환경에서 표준

### ✔ 예:

대형 소프트웨어, 기업/학교 배포형 앱

---

# 🎯 **2. 무설치(Portable) 실행 파일 배포**

압축을 풀면 바로 실행되는 방식 (EXE만 제공)

### ✔ 특징

* ZIP 다운로드 → 바로 실행
* 레지스트리·시스템 설치 없음
* USB에서도 실행 가능

### ✔ 예:

단일 유틸리티 프로그램, 개발 도구, 휴대용 앱

---

# 🎯 **3. 스토어 기반 배포 (Microsoft Store)**

Windows Store를 통한 공식 채널 배포

### ✔ 특징

* 자동 업데이트
* 설치/제거 안전성
* 신뢰성(바이러스 걱정 감소)
* 수익 배분 구조가 있음 (개발자 수익)

### ✔ 예:

UWP / WinUI3(MSIX) 기반 현대적 앱

---

# 🎯 **4. 기업용 배포 (Enterprise Deployment)**

조직 내 PC에 프로그램을 일괄 배포할 때 사용

### ✔ 방식

* SCCM(MEM) / Intune
* GPO 배포
* 오프라인 설치 패키지

### ✔ 특징

* IT 팀이 중앙에서 관리
* 특정 버전 강제 설치/업데이트 가능
* 인증/권한 관리 포함

---

# 🎯 **5. 웹 다운로드 + 자동 업데이트(Updater 포함)**

게임·툴 등에서 많이 사용

### ✔ 구성

* 메인 프로그램 EXE
* 자동 업데이트 프로그램(Launcher)
* 패치 서버

### ✔ 특징

* 첫 설치 후 계속 자동 패치
* 변경 파일만 빠르게 업데이트
* 사용자가 항상 최신 버전을 받음

---

# 🎯 **6. 구독 기반 / 계정 인증 기반 배포(라이선스 포함)**

소프트웨어의 설치 형태가 아니라 **판매 방식**에 따른 분류지만 실제 배포 패턴에도 영향이 큼.

### ✔ 방식

* 계정 로그인 방식
* 라이선스 키 입력
* 온라인 인증 + 클라우드 설정 동기화

### ✔ 특징

* 소프트웨어는 동일하지만 기능은 라이선스에 따라 활성화
* SaaS 방식과 혼합 가능
* 해킹·불법 복제를 줄이는 효과

### ✔ 예:

Photoshop, Office 365, AutoCAD 등

| 분류                     | 설명             | 대표 예          |
