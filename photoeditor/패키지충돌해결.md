## 💻 DEP0700 오류 해결 방법: 패키지 충돌

제공해주신 오류 코드 **DEP0700**과 메시지($0\text{x}80073\text{CFB}$)는 **Windows 앱 등록** 과정에서 충돌이 발생했음을 나타냅니다.

오류 메시지를 해석하면 다음과 같습니다:

> **"다른 사용자가 이미 이 앱의 패키지되지 않은 버전을 설치했습니다. 현재 사용자가 이것을 패키지된 버전과 바꿀 수 없습니다. 충돌을 일으키는 패키지는 96eb8b1a-8885-4cd2-ab4d-8f31ffc46bc7이며 CN=Microsoft Corporation에 의해 게시되었습니다."**

이는 현재 설치하려는 \*\*패키지된 버전(Packaged version)\*\*의 앱과 **충돌**하는 \*\*패키지되지 않은 버전(Unpackaged version)\*\*의 앱이 해당 장치에 **다른 사용자 계정**으로 이미 설치되어 있기 때문에 발생하는 문제입니다. 이 충돌을 해결하려면 해당 **충돌 패키지를 제거**해야 합니다.

-----

### 🛠️ 해결 단계

가장 일반적인 해결책은 충돌하는 **패키지되지 않은 앱**을 모든 사용자 계정에서 제거하는 것입니다. 이는 **PowerShell** 명령어를 통해 수행할 수 있습니다.

#### 1\. PowerShell 관리자 권한으로 실행

1.  **Windows 검색창**에 \*\*"PowerShell"\*\*을 입력합니다.
2.  검색 결과에서 \*\*"Windows PowerShell"\*\*을 **마우스 오른쪽 버튼으로 클릭**합니다.
3.  \*\*"관리자 권한으로 실행"\*\*을 선택합니다.

#### 2\. 충돌 패키지 제거 명령어 실행

PowerShell 창에 다음 명령어를 입력하고 **Enter** 키를 누릅니다. 이 명령어는 **모든 사용자**의 계정에서 해당 충돌 패키지를 제거합니다.

> **충돌 패키지 ID:** `96eb8b1a-8885-4cd2-ab4d-8f31ffc46bc7`

```powershell
Get-AppxPackage -AllUsers | Where-Object {$_.PackageFamilyName -like "96eb8b1a-8885-4cd2-ab4d-8f31ffc46bc7*"} | Remove-AppxPackage -AllUsers
```

  * **`Get-AppxPackage -AllUsers`**: 장치에 설치된 **모든 사용자**의 모든 앱 패키지를 가져옵니다.
  * **`Where-Object {...}`**: 가져온 패키지 중 \*\*패키지 패밀리 이름(PackageFamilyName)\*\*이 충돌 ID(`96eb8b1a-8885-4cd2-ab4d-8f31ffc46bc7`)로 시작하는 것을 필터링합니다.
  * **`Remove-AppxPackage -AllUsers`**: 필터링된 패키지를 **모든 사용자** 계정에서 제거합니다.

#### 3\. 앱 재등록 시도

제거 명령어를 실행한 후, 이전에 실패했던 **앱 등록/설치**를 다시 시도해 보세요. 문제가 해결되었을 가능성이 높습니다.

-----

### 🔍 참고: 충돌 패키지 확인

해당 패키지 ID는 Microsoft에서 게시한 **Xbox Identity Provider** 또는 이와 관련된 서비스 앱일 가능성이 높습니다. 만약 제거 후 다른 문제가 발생한다면, **Microsoft Store**에서 해당 앱을 다시 검색하여 설치하거나, `wsreset.exe` 명령어를 통해 Store 캐시를 초기화해 볼 수 있습니다.

혹시 제거 명령어 실행 후에도 동일한 오류가 발생한다면, 알려주시면 다른 해결책을 함께 찾아보겠습니다.
