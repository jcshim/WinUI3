# **산업에서 실제로 사용되는 배포 형태 6가지**

---

# ✅ **1. 소스 코드 형태(Source Code Licensing)**

고객에게 **전체 소스 코드**를 제공하는 방식.

### ✔ 장점

* 고객이 마음대로 수정/빌드 가능
* 내부 시스템에 맞게 커스터마이징 가능
* 고급 고객(대기업·연구소)에게 선호됨

### ✔ 단점

* 코드 유출 위험
* 가격이 가장 비싸며 계약서(NDA)가 반드시 필요

### ✔ 예

* 산업용 Vision 라이브러리 OEM
* 임베디드 장비용 SDK 커스터마이징

---

# ✅ **2. 정적 라이브러리(.lib) 판매**

개발자에게 **.lib 파일과 헤더(.h)** 를 제공하여
고객이 자신의 프로그램에 링크하여 사용할 수 있게 하는 방식.

### ✔ 특징

* 내부 구현은 노출되지 않음
* 고객은 단일 EXE 안에 전체 기능을 통합해 배포 가능
* 소스 공개 없이 기능 판매 가능

### ✔ 장점

* 매우 빠름(정적 링크)
* 소스 노출 없음
* 제어가 쉬움

### ✔ 단점

* 라이브러리를 사용하는 프로그램마다 재빌드 필요
* 특정 컴파일러 버전(VC++14, VC++17 등)과 ABI 의존성이 있음

### ✔ 예

OpenCV static build, Math/AI Engine Library 판매

---

# ✅ **3. 동적 라이브러리(DLL + 헤더) 판매**

가장 널리 사용되는 상업용 배포 방식.

### ✔ 구성

* DLL (기능 구현)
* .lib (import library)
* .h (API 선언)
* 문서(Documentation)

### ✔ 장점

* **소스코드 유출 없이 기능 제공**
* 업데이트가 쉬움(DLL만 교체하면 기능이 바뀜)
* 다양한 언어에서 불러 쓰기 가능(C++, C#, Python 등)

### ✔ 단점

* DLL 의존성 관리 필요
* 복잡한 API는 문서화 부담 증가

### ✔ 예

* OpenCV DLL
* TensorRT DLL
* 상용 영상처리 엔진 / 생체인식 알고리즘 DLL 판매
  (교수님의 Vision Lab 사업 형태와도 매우 잘 맞습니다)

---

# ✅ **4. SDK 형태(헤더 + DLL/Lib + 예제 + 문서)**

개발자가 사용할 수 있도록 **단순 DLL 판매를 넘어 SDK 패키지로 제공**하는 방식.

### ✔ 포함 요소

* API 문서
* C++/C#/Python 예제 코드
* 샘플 데이터
* 테스트 도구
* 라이선스 인증 모듈

### ✔ 장점

* 고객이 빠르게 사용할 수 있음
* 기술 지원 구조 확립
* 고가 판매가 가능(백만원~수천만원)

### ✔ 예

* Cognex Vision SDK
* NVIDIA Video/AI SDK
* Zebra Scanner SDK

---

# ✅ **5. 플러그인 형태 Plugin(.dll) 제공**

기존 플랫폼(Photoshop, 3DSMax, AfterEffects, Unity 등)에 기능을 추가하는 형태.

### ✔ 특징

* 사용자 프로그램에 기능 확장
* 독립 실행 프로그램이 아니라 "확장 기능"으로 판매
* 특정 인터페이스(API)를 준수해야 함

### ✔ 예

* Photoshop 필터 플러그인
* Unity Asset (DLL 기반)
* 산업용 머신비전 소프트웨어 확장 모듈

---

# ✅ **6. SaaS / API Key 기반(클라우드 기반 소프트웨어)**

로컬 프로그램 대신 **서버에서 기능을 제공 → API 호출로 사용**
AI 기능 제공 시 가장 널리 쓰임.

### ✔ 장점

* 소스 코드 100% 보호
* 월 수익 모델 가능
* 기능 업데이트 즉시 반영

### ✔ 예

* ChatGPT API
* OCR/Face Recognition API
* Vision Inspection Cloud API

---

# 📌 **정리: 판매 가능한 형태 전체 요약**

| 배포 형태                    | 제공 내용            | 소스 보호 | 난이도 | 산업 활용도        |
| ------------------------ | ---------------- | ----- | --- | ------------- |
| **Source Code**          | 전체 코드            | 낮음    | 중   | R&D, OEM      |
| **Static Library(.lib)** | .lib + .h        | 높음    | 중   | 엔진 판매         |
| **DLL + Header**         | .dll + .lib + .h | 매우 높음 | 중   | 가장 일반적        |
| **SDK 패키지**              | 라이브러리 + 문서 + 예제  | 매우 높음 | 높음  | 상용 SW         |
| **Plugin(.dll)**         | 기능 확장            | 높음    | 중   | Adobe/Unity 등 |
| **API/SaaS**             | 서버 기반 기능         | 최고    | 낮음  | 최신 AI 서비스     |

# 🎯 **교수님 연구·교육·사업 방향에서 가장 유리한 형태**

### 👉 **DLL + SDK 형태 판매**

(예: Vision Algorithm DLL + Header + 샘플)

이 방식은:

* 소스는 보호하고
* 기능은 제공하고
* 유지·업데이트가 쉽고
* 가격도 높게 받을 수 있어

**Vision Lab 기술(패턴인식, 3D, MTP 등)** 에 최적화된 판매 모델입니다.
