# 🔵 1. 패키지형(Packaged) 앱

## ✔ 개념

* **MSIX 패키지로 설치되는 앱**
* Windows가 앱을 “설치형 프로그램”으로 인식
* UWP에서 사용하던 API(스토리지, 파일, 알림 등) 대부분 사용 가능

## ✔ 특징

| 항목                   | 설명                           |
| -------------------- | ---------------------------- |
| **설치 방식**            | MSIX로 설치 (시스템에 등록)           |
| **권한/Capability**    | `Package.appxmanifest`에서 선언  |
| **파일 접근**            | UWP 스타일 Storage API 완전 사용 가능 |
| **알림(Notification)** | App Notification 사용 가능       |
| **자동 업데이트**          | MSIX가 지원                     |
| **시스템 통합**           | 시작 메뉴, 앱 목록 등에 자동 등록         |

## ✔ 장점

* Windows 기능과의 통합성이 매우 좋음
* API 제한이 거의 없음
* 배포 안정성 높음

## ✔ 단점

* 배포 과정이 MSIX라 약간 복잡할 수 있음
* 기업/학교 PC에서는 MSIX 설치가 제한될 수 있음

# 🟠 2. 언패키지형(Unpackaged) 앱

## ✔ 개념

* **EXE로 그대로 실행하는 전통적인 Win32 스타일 앱**
* 설치 없이 “실행 파일만”으로 동작
* Windows App SDK를 초기화하여 WinUI3 UI만 사용하는 형태

## ✔ 특징

| 항목                   | 설명                                 |
| -------------------- | ---------------------------------- |
| **설치 방식**            | 설치 없이 exe로 바로 실행                   |
| **권한/Capability**    | manifest 기반(기본 Win32)              |
| **파일 접근**            | 전통 Win32 파일 API 또는 제한된 Storage API |
| **알림(Notification)** | 일부 기능 제한됨                          |
| **MSI/Zip 배포**       | 간단하게 배포 가능                         |

## ✔ 장점

* 학생 실습/연구용으로 배포가 매우 쉬움
* 기존 Win32 C++ 코드와 연결이 편함
* 기업 내부에서 설치 없이 빠르게 테스트 가능

## ✔ 단점

* Windows 기능(UWP 스타일 API) 일부가 사용 불가
* 시스템과의 통합(앱 인스턴스 관리, 알림 등)이 제한됨
* 앱 업데이트 자동화 기능 부재

# 🟢 3. 가장 큰 차이만 요약하면

| 구분                      | 패키지형 (MSIX)                | 언패키지형 (EXE)     |
| ----------------------- | -------------------------- | --------------- |
| **앱 등록 방식**             | Windows에 설치됨               | 설치 없이 실행        |
| **API 사용 범위**           | Windows Runtime API 대부분 가능 | 일부 API 제한됨      |
| **파일/스토리지 권한**          | 앱 매니페스트로 제어                | Win32 권한 기반     |
| **배포**                  | MSIX 패키지 필요                | ZIP, exe로 바로 배포 |
| **앱 ID/App Activation** | 가능                         | 대부분 지원 안 됨      |
| **업데이트 방식**             | 자동화(MSIX)                  | 직접 배포해야 함       |

---

# 🧠 교수님 수업 기준으로 추천

* **수업 초반 / 실습 위주 / 빠른 반복**
  → **Unpackaged 앱이 훨씬 편함**
  → EXE로 바로 실행되므로 학생들이 오류 없이 따라갈 수 있음

* **수업 후반 / 앱 완성 / 배포까지 경험시키고 싶다면**
  → **Packaged(MSIX)** 로 한 번 빌드해서 설치·업데이트까지 보여주면 최적

# 🎯 한 줄 요약
