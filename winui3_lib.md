# ✔ Visual Studio에서 WinUI 3 앱에 가장 잘 맞는 라이브러리 템플릿 3가지

아래 세 가지 중에서 **용도에 맞게 선택**하는 것이 정답입니다.

---

# 1️⃣ **C++ 정적 라이브러리 (Static Library .lib)** → 가장 많이 사용됨

### 템플릿 이름

**C++ → Static Library (Windows)**

### 언제 사용?

* 알고리즘, 수학, 영상처리, 인식 엔진처럼
  **UI가 없는 순수 C++ 로직을 WinUI 3 앱에서 재사용할 때**
* 교수님의 Vision Lab 과제, 숫자 인식 엔진, 데이터 처리 등

### 장점

* EXE에 완전히 링크되어 배포가 간단
* DLL처럼 추가 파일 없이 깔끔
* 가장 빠르고 안정적

### WinUI 3 앱에서 사용

* 프로젝트 → 우클릭 → “참조 추가” → .lib 링크하면 끝

👉 **수업/연구 프로젝트에서 가장 추천되는 방식**입니다.

---

# 2️⃣ **C++ DLL (Dynamic Library, .dll + .lib)** → 외부 판매/재사용 목적일 때

### 템플릿 이름

**C++ → Windows Desktop DLL**

### 언제 사용?

* 라이브러리를 **외부 업체/고객에게 판매**하고 싶을 때
* C#, Python 등 다른 언어에서도 사용되게 하려면
* “SDK 형태로 배포"하고 싶은 경우

### 장점

* 소스코드를 숨길 수 있음
* 다른 앱에서도 쉽게 사용 가능
* 업데이트 시 DLL만 교체 가능

👉 **상용 라이브러리 판매 또는 연구실 엔진 배포 시 최적**

---

# 3️⃣ **Windows Runtime Component (C++/WinRT)** → WinRT API로 만들고 싶을 때

### 템플릿 이름

**C++/WinRT Windows Runtime Component**

### 언제 사용?

* 라이브러리를 **WinRT 클래스(API)** 형태로 노출하고 싶을 때
* WinUI 3 **C# 앱과도 공유하여 사용**하고 싶을 때
* XAML에서 직접 바인딩 가능한 API를 만들고 싶다면

### 장점

* WinRT 객체처럼 new 가능
* C# WinUI 3 앱에서도 그대로 사용 가능
* XAML Binding 가능

📌 **단점**: 일반 라이브러리보다 구조가 복잡함

---

# ❌ 사용하면 안 되는 템플릿

### **정적 라이브러리(유니버설 Windows)**

* UWP 앱용
* WinUI 3(Win32 데스크톱 앱)과 타깃이 일치하지 않음
* 링크 오류 또는 SDK mismatch 발생

👉 **WinUI 3 개발에는 맞지 않음. 절대 선택하지 마세요.**

---

# 📌 어떤 것을 선택해야 할까? (교수님 기준 추천)

| 상황                          | 추천 템플릿                                    |
| --------------------------- | ----------------------------------------- |
| WinUI3 수업에서 공용 로직 만들기       | **Static Library (.lib)**                 |
| 연구실 Vision/AI 엔진 → 외부 판매/공유 | **DLL + Header**                          |
| C# WinUI3 앱에서도 같은 컴포넌트 사용   | **Windows Runtime Component (C++/WinRT)** |
| 단일 WinUI3 앱만 사용하는 간단 라이브러리  | **Static Library**                        |

* 학생들이 실습하기 가장 쉽고
