## 🧩 1단계 — AI 기본 개념만 알기

**목표:** “AI가 이미 학습된 모델을 이용해 판단한다” 정도만 이해하기.

* **AI**: 데이터를 보고 규칙을 스스로 학습한 프로그램.
* **MNIST**: 손으로 쓴 숫자(0~9) 이미지 7만 장짜리 대표 데이터셋.
* **모델(model)**: 숫자 이미지를 보고 어떤 숫자인지 맞추도록 학습된 “두뇌 파일”.
* **ONNX**: 그 두뇌를 저장한 “표준 파일 형식(예: mnist-8.onnx)”.

👉 **정리:** AI가 이미 학습된 모델을 이용해 추론(inference)만 수행함.
즉, “학습(training)”은 이미 끝난 모델을 받아서 “예측(prediction)”만 하는 겁니다.

---

## ⚙️ 2단계 — ONNX와 ONNX Runtime 이해하기

**목표:** “AI 모델 파일을 실행하려면 ORT를 쓴다” 정도만 알기.

* **ONNX**: AI 모델을 담은 파일 (예: `mnist-8.onnx`).
* **ONNX Runtime (ORT)**: 그 파일을 실제로 돌려주는 “엔진”.
  → C++ 라이브러리 형태로 제공, 입력을 넣으면 결과를 돌려줍니다.

👉 **정리:** WinUI 앱에서 숫자를 인식하려면
→ “ONNX Runtime으로 mnist-8.onnx를 불러서”
→ “내가 그린 이미지를 입력으로 넣고 결과(0~9)를 받는다”.

---

## 🧑‍💻 3단계 — WinUI 3 C++/WinRT 기본 익히기

**목표:** UI 만들고, 버튼 클릭·그리기 이벤트 처리하는 법 익히기.

공부 항목:

1. **XAML 기본**: 버튼, 슬라이더, 캔버스.
2. **이벤트 연결**: `Click="RecognizeButton_Click"`.
3. **Canvas에서 그림 그리기**: PointerPressed / Moved / Released.
4. **RenderTargetBitmap**으로 그린 그림을 이미지로 가져오기.

👉 교수님은 이미 이 단계는 거의 완료하셨습니다 👍

---

## 🧮 4단계 — 이미지 전처리(Preprocessing)

**목표:** 그린 그림을 모델 입력으로 변환하기.

MNIST 모델은 **28×28 크기의 흑백(1채널)** 이미지를 입력으로 받습니다.
그래서 해야 할 일은:

1. Canvas를 `RenderTargetBitmap`으로 캡처.
2. 28×28 크기로 축소.
3. 흑백 변환 (R,G,B → 밝기 0~1).
4. 배경/글씨 색 반전 (흑배경+백글씨로 맞추기).
5. 1×1×28×28 float 배열로 변환.

👉 “이미지 → 숫자 배열”이 되면 ORT에 넣을 수 있습니다.

---

## 🧠 5단계 — ONNX Runtime으로 추론하기

**목표:** 모델을 로드하고 결과(0~9) 얻기.

절차 요약:

1. `Ort::Env`, `Ort::Session`으로 모델 로드.
2. 입력 텐서(`float[1][1][28][28]`)를 만들고 Run 실행.
3. 출력(`float[1][10]`) 중 가장 큰 값을 argmax로 숫자 판단.
4. 화면에 “인식 결과: 7” 이런 식으로 출력.
